Why Use Version Control?

History

Version Control Software keeps track of every time your project updates. It allows you to track the evolution of the project and see what everybody else has done.

Collaboration

Version Control allows you to keep your project up-to-date between users. When you change a file, you push those changes to a common repository. Other users can then pull those change and make changes of their own.

Backup

With Version Control, you have access to every version of the project that's been committed. You can easily revert to a previous version if something breaks.

Learn Git

We've only scratched the surface of what you cand do with Git. If you want to learn more, these sources are a great place to start.

Atlassian provides has some beautiful git documentation that is great for beginners.

If you want to learn about Git's more advanced features, Pro Git covers just about everything.

Github is a site that promotes social coding. Users can host git projects online where others can view it and contribute.

Bitbucket is similar to Github, but all repositories are private for teams of 5 or less. Great for small projects.

TryGit is a simple tutorial that teaches you the basics of git.

GitLab is another site that hosts git repositories. User can host it on GitLab, or download the tools to make their own git servers.

Find Out More

Want to learn about version control systems other than Git? try these!

Mercurial

A distributed version control software that works similarly to git. Mercurial repositories are supported on Bitbucket!

Subversion

A widely used centralized version control software.

Perforce

Enterprise-level version control that handles assets and large files. Used widely in the game industry.

Team Foundation Server

A suite of version control and project management tools by Microsoft. Integrated with Visual Studio.
